# ESPECIFICAÇÕES DETALHADAS V2

## 1. Internacionalização (i18n)
- Finalizar implementação de EN, ES, FR.
- Todas as strings da interface devem estar no sistema de tradução.
- O idioma deve ser carregado:
  • pela escolha do usuário  
  • persistido localmente  
  • default: seguir sistema na primeira abertura

---

## 2. Temas (Sistema / Light / Dark)
- Criar 3 temas completos.
- O tema “Sistema” deve seguir `Appearance.getColorScheme()` em tempo real.
- Temas precisam aplicar:
  • cores  
  • botões  
  • backgrounds  
  • inputs  
  • textos  
- Persistir escolha do usuário localmente.

---

## 3. Persistência Local
Use **AsyncStorage** (ou equivalente) para salvar:

- Nome do usuário
- Avatar escolhido
- Idioma selecionado
- Tema selecionado
- Preferências de som
- Preferências de vibração

Requisitos:
- Dados devem carregar instantaneamente ao abrir o app.
- Se o app for apagado, tudo deve ser apagado naturalmente.

---

## 4. Perfil Reformulado
- Adicionar **6 avatares masculinos e 6 femininos**, estilo minimalista.
- Layout de seleção com:
  • borda de seleção  
  • efeito de escala ao tocar  
  • avatar em grid  
- Input de nome com:
  • placeholder traduzido  
  • validação simples (+ feedback visual)

---

## 5. Inputs de Tempo (Melhorados)
Cada campo (preparação / exercício / descanso / rounds):

- Controles de +5 / –5 (como já existe)
- **Campo editável manualmente**
- Aceitar:
  • segundos (ex: “45”)  
  • minutos e segundos (ex: "1:30")  
  • valores quebrados (ex: “7.5s” ou “1.2m”)  

Regras:
- Converter para segundos no estado interno.
- Validar valores negativos.
- Atualizar estimativa total em tempo real.

---

## 6. UI/UX (Melhorias)
- Manter o design atual, mas aprimorar:
  • transições suaves entre telas  
  • botões com animação de toque (scale + haptic)  
  • inputs com highlight  
- Ajustar a tela do timer para exibir mais contraste dependendo do tema.
- Add microanimação no momento de transição de fase.
